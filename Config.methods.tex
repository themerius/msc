\renewcommand{\abstract}{
Aktuelle Textverarbeitungssysteme weisen beim Erstellen von wissenschaftlichen Texten
noch Defizite auf:
Dokumente haben keine Kenntnisse über das Wissensfeld, welches sie beschreiben.
Das fördert Inkonsistenzen im Text.
Bedeutungen einzelner Bestandteile eines Dokuments sind dem System nicht bewusst,
jeder Autor muss sich erneut um den formal korrekten Aufbau des Dokuments kümmern.
In dieser Arbeit ist ein browserbasierter prototypischer Dokumenteditor entstanden,
der diese Defizite abfedert und anhand drei konrekter Anwendungsfälle die Praxistauglichkeit prüft.
Daraus sind allgemeine Theorien über Dokumente entstanden:
Ein natürliches Modell für Dokumente ist der Graph bzw. abstrakte Syntaxbaum.
Dokumentelemente (z.B. Absätze, Abbildungen, etc.) sind die einzelnen Bestandteile eines Dokuments und werden als Graph-Knoten abgebildet.
Dokumentelemente kapseln ein Modell einer spezifischen Domäne.
Es gibt eine allgemeingültige Taxonomie für den inneren Aufbau von Dokumenten.
Dadruch kann eine Dokumentklasse formal spezifiziert werden, worin
Vorgaben über erlaube Abfolgen und Verschachtelungen von Dokumentelementen
enthalten sind.
Dies fungiert als ein passendes Metamodell.
}
